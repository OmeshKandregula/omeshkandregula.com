Summary

A tool that correlates alerts, logs, traces, and deploys in real time — then gives on-call engineers a structured, timestamped incident view with AI-generated root cause hypotheses. Not a chatbot. A workflow.

The Problem

When something breaks at 2 AM, the most expensive part isn't the fix — it's finding the signal in the noise. Engineers scramble across Slack, Datadog, CloudWatch, PagerDuty, and deployment logs with no structured view of what's happening and when. Time to root cause is the real cost.

  • On-call engineers context-switch across 5+ tools during active incidents.
  • Alert timelines and deploy timelines are never in the same view.
  • AI chat tools hallucinate root causes when context is missing.
  • Postmortems are written from memory hours after the incident — inaccurate by design.

Context

Builder & Product Lead 36-hour Hackathon, 2025

Designed and built the full prototype in 36 hours. Schema design, signal correlation engine, dashboard, and AI layer — all shipped within the window. The constraint was the point.

What I Built

Incident correlation engine with structured dashboard and AI hypothesis layer.

  • Signal ingestion across 4 types: alerts, logs, traces, and deploys — correlated by timestamp
  • Deterministic schema enforcement: every incident entry is structured, queryable, and diff-able
  • Incident timeline dashboard replacing the 5-tab Slack-Datadog-CloudWatch scramble
  • AI hypothesis layer that generates root cause candidates only after deterministic context is assembled

Outcomes

Full prototype in 36 hours

Schema, correlation engine, dashboard, and AI layer — all built and demo-ready within the hackathon window.

Zero hallucination architecture

AI only activates after structured signal context is assembled. No guessing from empty input.

Automatic postmortem draft

Structured timeline data generates a postmortem draft in real time — written while the incident is happening, not reconstructed after.

Key Decisions

Data Model

Schema enforcement first

Every signal ingested must conform to a strict schema before it enters the system. Garbage in, garbage out — especially with AI on top.

UX

Dashboard over chat

Incident response is spatial. A chatbot interface forces you to describe what you're seeing. A dashboard shows it. Spatial beats sequential under pressure.

AI Layer

No vector DB

Retrieval-augmented incident analysis sounds clever but adds latency, complexity, and hallucination risk. Structured context passed directly to the model is more reliable.

Architecture

Deterministic layers

Correlation, deduplication, and timeline construction are deterministic. AI only touches the hypothesis layer. Each layer has a clear contract with the next.

Key Takeaway

AI is not the product. Structure is. The model can only be as good as the context you give it. Deterministic layers make AI trustworthy — not the other way around.
